Сперва я составила вот такой план:

1. Определение требований

• Функциональные требования:
  – Обработка POST-запросов для операций DEPOSIT и WITHDRAW;
  – Обработка GET-запросов для получения баланса кошелька.
• Нефункциональные требования:
  – Обработка ошибок;
  – Формат ответов для заведомо неверных запросов.

2. Настройка окружения

Установить/обновить версию ide, JDK, Docker, Docker Compose и пр.

3. Создание проекта

Создать новый проект Spring Boot с использованием Spring Initializr, выбрав зависимости: Spring Web, Spring Data JPA, PostgreSQL Driver.

4. Настройка БД

• Создать Dockerfile для PostgreSQL;
• Создать docker-compose.yml для настройки контейнеров

5. Создание модели данных

• Создать класс Wallet с полями id, balance и методами для управления балансом;
• Реализовать репозиторий WalletRepository.

6. Реализация бизнес-логики

• Создать сервис WalletService, который будет обрабатывать операции deposit и withdraw;
• Реализовать валидацию на наличие кошелька, достаточность средств и пр.

7. Реализация контроллера

Создать контроллер WalletController для обработки запросов на получение и изменение баланса

8. Обработка ошибок

Реализовать обработку ошибок с помощью @ControllerAdvice для возврата сообщений об ошибке при возникновении исключений

9. Написание тестов

Я недостаточно шарю за TDD, чтобы делать это вместе с предыдущими шагами, поэтому попытаюсь написать юниты на этом этапе

10. Докеризация приложения

Создать Dockerfile для приложения  

11. Запуск приложения

Добиться, чтобы команда “docker-compose up --build” отработала без ошибок, и было запущено два контейнера: один для БД, другой - для приложения

12. Мини-тестирование API

Покидать запросы через Postman

13. Документация и завершение проекта
